> 本文档定义 HarnessGenJ 框架如何严格实现 JVM 的三大核心机制

## 一、JVM 内存模型 → Agent 记忆模型

### JVM 内存区域划分

```
┌─────────────────────────────────────────────────────────────────┐
│                        JVM 内存模型                              │
├─────────────────────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐          │
│  │   方法区     │  │     堆       │  │     栈       │          │
│  │ (Method Area)│  │   (Heap)     │  │  (Stack)     │          │
│  │              │  │              │  │              │          │
│  │ 类信息       │  │ 对象实例     │  │ 方法栈帧     │          │
│  │ 常量池       │  │ GC管理       │  │ 局部变量     │          │
│  │ 静态变量     │  │ 分代收集     │  │ 操作数栈     │          │
│  └──────────────┘  └──────────────┘  └──────────────┘          │
│                                                                 │
│  ┌──────────────┐  ┌──────────────┐                            │
│  │ 程序计数器   │  │ 本地方法栈   │                            │
│  │ (PC Register)│  │(Native Stack)│                            │
│  │ 执行位置     │  │ JNI调用      │                            │
│  └──────────────┘  └──────────────┘                            │
└─────────────────────────────────────────────────────────────────┘
```

### Agent 记忆模型严格映射

| JVM 区域 | Agent 记忆区 | 存储内容 | 管理策略 |
|----------|-------------|----------|----------|
| **方法区** | `MetaSpace` | Agent定义、工具规范、系统知识 | 永久存储，不回收 |
| **堆 (Heap)** | `MemoryHeap` | 对话历史、执行结果、大数据 | 分代GC，自动回收 |
| **栈 (Stack)** | `ExecutionStack` | 当前任务栈帧、局部上下文 | 任务完成自动弹出 |
| **程序计数器** | `ProgressCounter` | 当前执行步骤、任务进度 | 实时更新 |
| **本地方法栈** | `ToolCallStack` | 外部工具/API调用记录 | 调用完成清理 |

## 二、JVM 分代GC → 分代记忆回收

### JVM 堆分代结构

```
┌─────────────────────────────────────────────────────────────────┐
│                        JVM Heap 分代                             │
├─────────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                 Young Generation (新生代)                │   │
│  │  ┌─────────┐  ┌─────────┐  ┌─────────┐                  │   │
│  │  │  Eden   │  │Survivor0│  │Survivor1│                  │   │
│  │  │ 新对象  │  │  存活1  │  │  存活2  │                  │   │
│  │  └─────────┘  └─────────┘  └─────────┘                  │   │
│  │        ↓ Minor GC (频繁, 快速)                           │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              ↓ 晋升 (年龄达标)                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │              Old Generation (老年代)                      │   │
│  │  │        长期存活对象, 大对象直接分配        │          │   │
│  │  └─────────────────────────────────────────────────────┘   │
│  │        ↓ Major GC / Full GC (较少, 较慢)                  │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              ↓ 晋升 (极度重要)                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │              Permanent Generation (永久代)                │   │
│  │  │     类信息, 常量池, 永不回收        │                  │   │
│  │  └─────────────────────────────────────────────────────┘   │
│  └─────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

### Agent 记忆分代映射

| JVM 分代 | Agent 记忆分代 | 存储内容 | 回收策略 |
|----------|---------------|----------|----------|
| **Eden区** | `EdenMemory` | 最新对话消息 | 每轮对话后快速清理 |
| **Survivor区** | `SurvivorMemory` | 短期重要消息 | 晋升阈值检测 |
| **Old区** | `OldMemory` | 长期对话历史 | 满时压缩/摘要 |
| **Permanent区** | `PermanentMemory` | 核心知识、Agent定义 | 永不回收 |

### GC算法映射

| JVM GC算法 | Agent 记忆回收算法 | 实现方式 |
|------------|-------------------|----------|
| **Mark-Sweep** | `标记-清除` | 标记无用消息，直接删除 |
| **Copying** | `复制算法` | Eden→Survivor复制存活消息 |
| **Mark-Compact** | `标记-压缩` | Old区消息压缩为摘要 |
| **G1 (Garbage First)** | `优先回收` | 优先回收低价值记忆区 |

## 三、JVM JIT编译 → 热点自动装配

### JIT 编译流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        JIT 编译流程                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   方法调用 ─→ 解释执行 ─→ 热点检测 ─→ JIT编译 ─→ 优化执行      │
│      │            │            │            │            │      │
│      ↓            ↓            ↓            ↓            ↓      │
│   字节码       计数器计数    调用阈值     编译为机器码   内联优化 │
│   解释器       方法计数器    达到阈值     本地代码缓存   逃逸分析 │
│                                                                 │
│   ┌──────────────────────────────────────────────────────┐     │
│   │              编译优化策略                              │     │
│   │  • 方法内联 (Method Inlining)                         │     │
│   │  • 逃逸分析 (Escape Analysis)                         │     │
│   │  • 循环优化 (Loop Optimization)                       │     │
│   │  • 死代码消除 (Dead Code Elimination)                 │     │
│   └──────────────────────────────────────────────────────┘     │
└─────────────────────────────────────────────────────────────────┘
```

### Agent 热点自动装配映射

| JIT 机制 | Agent 热点机制 | 实现方式 |
|----------|---------------|----------|
| **热点检测** | `HotspotDetector` | 统计工具/知识调用频率 |
| **编译阈值** | `AssemblyThreshold` | 达到阈值触发自动装配 |
| **方法内联** | `KnowledgeInlining` | 高频知识嵌入执行流程 |
| **逃逸分析** | `ContextEscapeAnalysis` | 分析上下文依赖范围 |
| **代码缓存** | `AssemblyCache` | 缓存已装配的执行模板 |

## 五、工作流程示例

### 记忆回收流程 (类似GC)

```
┌─────────────────────────────────────────────────────────────────┐
│                     记忆回收流程                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 新消息进入 EdenMemory                                       │
│     │                                                           │
│     ↓                                                           │
│  2. Eden满时触发 Minor GC                                       │
│     │  ┌─────────────────────────────────┐                     │
│     │  │ • 标记: 分析消息重要性           │                     │
│     │  │ • 复制: 存活消息 → Survivor      │                     │
│     │  │ • 清除: Eden清空                 │                     │
│     │  └─────────────────────────────────┘                     │
│     ↓                                                           │
│  3. Survivor消息年龄增长                                        │
│     │                                                           │
│     ↓ (年龄 > MaxTenuringThreshold)                            │
│  4. 晋升到 OldMemory                                            │
│     │                                                           │
│     ↓ (Old满时)                                                 │
│  5. 触发 Major GC / Full GC                                     │
│     │  ┌─────────────────────────────────┐                     │
│     │  │ • 标记: 标记Old区存活内容        │                     │
│     │  │ • 压缩: 历史压缩为摘要           │                     │
│     │  │ • 清除: 删除无用内容             │                     │
│     │  └─────────────────────────────────┘                     │
│     ↓                                                           │
│  6. 极重要内容晋升到 PermanentMemory                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 热点自动装配流程 (类似JIT)

┌─────────────────────────────────────────────────────────────────┐

## 六、关键参数设计

### GC 参数 (类似 JVM GC 参数)

| JVM 参数 | Agent 参数 | 说明 |
|----------|-----------|------|
| `-Xms` | `initial_memory_size` | 初始记忆容量 |
| `-Xmx` | `max_memory_size` | 最大记忆容量 |
| `-Xmn` | `young_memory_size` | 新生代大小 |
| `MaxTenuringThreshold` | `promotion_threshold` | 晋升阈值 |
| `SurvivorRatio` | `survivor_ratio` | Eden:Survivor比例 |
| `PretenureSizeThreshold` | `large_message_threshold` | 大消息直接进Old |

### JIT 参数 (类似 JVM JIT 参数)

| JVM 参数 | Agent 参数 | 说明 |
|----------|-----------|------|
| `-XX:CompileThreshold` | `assembly_threshold` | 装配阈值 |
| `-XX:MaxInlineSize` | `max_inline_size` | 最大内联大小 |
| `-XX:InlineFrequency` | `inline_frequency` | 内联频率阈值 |

## 七、实现优先级

| 优先级 | 功能 | JVM对应 | 说明 |
|--------|------|---------|------|
| **P0** | 分代记忆管理 | Heap分代 | 核心架构 |
| **P0** | 记忆回收机制 | GC算法 | 核心功能 |
| **P1** | 分模块记忆区 | 内存区域 | 结构完善 |
| **P1** | 热点检测 | JIT热点检测 | 性能优化基础 |
| **P2** | 自动装配 | JIT编译 | 高级优化 |
| **P2** | 装配缓存 | Code Cache | 缓存机制 |
