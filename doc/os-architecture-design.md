### 1.2 角色定义

| 角色 | 操作系统对应 | 职责 | 数量 |
|------|-------------|------|------|
| **Scheduler** | 进程调度器 | 任务调度、资源分配、负载均衡 | 1个 (全局唯一) |
| **Producer** | 生产者进程 | 任务创建、任务分解、任务发布 | 多个 (Manager/Planner等) |
| **Consumer** | 消费者进程 | 任务执行、结果上报 | 多个 (Worker Agent) |
| **MemoryManager** | 内存管理器 | JVM风格记忆管理 | 1个 (全局唯一) |
| **TaskKernel** | 内核 | 任务生命周期管理、状态机 | 1个 (全局唯一) |

## 二、任务生命周期闭环

### 2.1 任务状态机 (类似进程状态)

```
                    ┌─────────────────────────────────────┐
                    │           TASK LIFECYCLE            │
                    └─────────────────────────────────────┘

    ┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
    │ CREATED │────→│  READY  │────→│ RUNNING │────→│  DONE   │
    │  创建   │     │  就绪   │     │  执行   │     │  完成   │
    └─────────┘     └─────────┘     └─────────┘     └─────────┘
         │               │               │               │
         │               │               │               │
         │               │               ↓               │
         │               │         ┌─────────┐          │
         │               │         │ BLOCKED │          │
         │               │         │  阻塞   │          │
         │               │         └─────────┘          │
         │               │               │               │
         │               │               ↓               │
         │               └───────────────┴──────────────→│
         │                               │               │
         │                               ↓               │
         │                         ┌─────────┐          │
         └────────────────────────→│ FAILED  │←─────────┘
                                   │  失败   │
                                   └─────────┘
```

### 2.2 状态转换规则

| 当前状态 | 触发事件 | 目标状态 | 执行者 |
|----------|----------|----------|--------|
| CREATED | 任务提交 | READY | Producer |
| READY | 调度执行 | RUNNING | Scheduler |
| RUNNING | 执行完成 | DONE | Consumer |
| RUNNING | 等待资源 | BLOCKED | Consumer |
| RUNNING | 执行失败 | FAILED | Consumer |
| BLOCKED | 资源就绪 | READY | Scheduler |
| FAILED | 重试 | READY | Scheduler |
| FAILED | 放弃 | DONE (失败) | Scheduler |

### 2.3 闭环管理流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         TASK CLOSED-LOOP MANAGEMENT                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. TASK CREATION (生产阶段)                                                │
│     ┌──────────┐                                                            │
│     │ Producer │ ──创建任务──→ TaskQueue (CREATED/READY)                    │
│     │  Agent   │                - 任务ID分配                                │
│     └──────────┘                - 初始状态设置                              │
│                                - 依赖关系建立                               │
│                                                                             │
│  2. TASK SCHEDULING (调度阶段)                                              │
│     ┌──────────┐                                                            │
│     │Scheduler │ ──选择任务──→ 分配给空闲Consumer                           │
│     │          │                - 优先级调度                                │
│     └──────────┘                - 负载均衡                                  │
│                                - 状态: READY → RUNNING                      │
│                                                                             │
│  3. TASK EXECUTION (消费阶段)                                               │
│     ┌──────────┐                                                            │
│     │ Consumer │ ──执行任务──→ 结果写入MemoryManager                        │
│     │  Agent   │                - 执行日志记录                              │
│     └──────────┘                - 状态更新: RUNNING → DONE/FAILED           │
│                                                                             │
│  4. TASK COMPLETION (闭环阶段)                                              │
│     ┌──────────┐                                                            │
│     │  Kernel  │ ──验证闭环──→ 任务生命周期结束                             │
│     │          │                - 结果验证                                  │
│     └──────────┘                - 资源回收                                  │
│                                - 通知Producer                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 三、核心组件设计

### 3.1 Scheduler (调度器)

```python
class Scheduler:
    """
    任务调度器 - 类似操作系统进程调度器

    职责:
    1. 任务调度: 从READY队列选择任务执行
    2. 资源分配: 分配Consumer给任务
    3. 负载均衡: 均衡Consumer负载
    4. 优先级管理: 按优先级调度

    调度算法:
    - FIFO: 先进先出
    - PRIORITY: 优先级调度
    - ROUND_ROBIN: 时间片轮转
    - FAIR: 公平调度
    """
```

### 3.2 Producer (生产者)

```python
class Producer:
    """
    任务生产者 - 创建和发布任务

    角色:
    - ManagerAgent: 管理型生产者，接收用户请求，分解任务
    - PlannerAgent: 规划型生产者，制定执行计划
    - CoordinatorAgent: 协调型生产者，分配子任务

    职责:
    1. 任务创建: 定义任务内容、优先级、依赖
    2. 任务分解: 将复杂任务分解为子任务
    3. 任务发布: 将任务提交到队列
    4. 结果接收: 接收任务完成通知
    """
```

### 3.3 Consumer (消费者)

```python
class Consumer:
    """
    任务消费者 - 执行任务

    角色:
    - WorkerAgent: 工作型消费者，执行具体任务
    - SpecialistAgent: 专家型消费者，执行特定类型任务

    职责:
    1. 任务领取: 从队列领取任务
    2. 任务执行: 执行任务逻辑
    3. 结果上报: 将结果写入MemoryManager
    4. 状态更新: 更新任务状态
    """
```

### 3.4 TaskKernel (任务内核)

```python
class TaskKernel:
    """
    任务内核 - 管理任务生命周期

    职责:
    1. 任务注册: 注册新任务，分配ID
    2. 状态管理: 管理任务状态机
    3. 依赖解析: 解析任务依赖关系
    4. 闭环验证: 确保任务闭环完成
    5. 资源回收: 清理已完成任务
    """
```

### 3.5 TaskQueue (任务队列)

```python
class TaskQueue:
    """
    任务队列 - 类似操作系统进程队列

    队列分区:
    - ready_queue: READY状态任务
    - running_queue: RUNNING状态任务
    - blocked_queue: BLOCKED状态任务
    - done_queue: DONE状态任务 (历史)
    - failed_queue: FAILED状态任务

    特性:
    - 优先级队列: 按优先级排序
    - 延迟队列: 延迟执行任务
    - 死信队列: 失败任务处理
    """
```

# 任务创建
task = producer.create_task(
    name="research_task",

    priority=Priority.HIGH,

# 任务发布
task_id = producer.publish_task(task)

consumer = scheduler.schedule_next()

result = await consumer.execute(task)

consumer.report_result(task_id, result)

## 七、与JVM记忆管理的协作

| 场景 | MemoryManager操作 | Task操作 |
|------|-------------------|----------|
| 任务创建 | 记录创建日志到Eden | 状态: CREATED |
| 任务执行 | 执行过程存入Eden | 状态: RUNNING |
| 任务完成 | 重要结果晋升到Old | 状态: DONE |
| 任务失败 | 错误信息存入Eden | 状态: FAILED |
| 核心知识 | 存入Permanent | 用于任务决策 |
| GC触发 | 清理过期任务数据 | 释放资源 |
